# 🚀 Инструкция по запуску WebAR приложения

## Требования

- **Браузер:** Chrome, Firefox, Edge (версия 90+) или Safari на iOS 14.5+
- **Камера:** Веб-камера (встроенная или USB)
- **Интернет:** Для загрузки библиотек с CDN
- **HTTPS:** В продакшене требуется защищенное соединение

## 🏃 Локальный запуск

### Способ 1: Использование Python (рекомендуется)

```bash
# Python 3
cd /path/to/byd
python -m http.server 8000

# или Python 2
python -m SimpleHTTPServer 8000
```

Затем откройте в браузере: **http://localhost:8000**

### Способ 2: Использование Node.js

```bash
# Установить простой HTTP сервер
npm install -g http-server

# Запустить
cd /path/to/byd
http-server
```

Откройте: **http://localhost:8080**

### Способ 3: Использование VS Code Live Server

1. Установите расширение "Live Server" в VS Code
2. Кликните правой кнопкой на `index.html` → "Open with Live Server"

## 🐛 Отладка

### Если камера не работает

**Шаг 1:** Проверьте консоль браузера (F12)
```
❌ Признаки:
- NotAllowedError - разрешение отклонено
- NotFoundError - камера не найдена
- NotReadableError - камера занята другим приложением
```

**Шаг 2:** Проверьте разрешения
```bash
# На Linux
ls /dev/video*  # Должно быть /dev/video0 или /dev/video1

# Убедитесь что браузер имеет доступ
```

**Шаг 3:** Перезагрузите страницу и дайте разрешение заново

### Если модели не видны

1. **Проверьте консоль:**
   ```
   ✓ Model yüklendi: cap1  # Модель загружена
   обнаружено лиц=1        # Лицо обнаружено
   видна=true              # Модель видна
   ```

2. **Если "обнаружено лиц=0":**
   - Убедитесь что ваше лицо видно на камере
   - Улучшите освещение
   - Установите камеру на расстояние 30-100 см

3. **Если модель загружается медленно:**
   - Проверьте скорость интернета
   - Откройте вкладку Network в консоли для мониторинга

### Быстрая диагностика

Скопируйте в консоль браузера для проверки статуса:

```javascript
// Проверка инициализации
console.log("=== ДИАГНОСТИКА ===");
console.log("Scene дети:", scene.children.length);
console.log("Активная модель:", activeKey);
console.log("Current model:", currentModel);
console.log("Видео:", video.videoWidth, "x", video.videoHeight);
console.log("Canvas:", renderer.domElement.width, "x", renderer.domElement.height);
```

## 🎨 Тестирование моделей

### Параметры моделей (MODELS_CONFIG)

```javascript
const MODELS_CONFIG = {
    cap1: {
        type: 'cap',           // Тип: cap (шапка) или glasses (очки)
        url: './cap1.glb',     // Путь к файлу
        scale: 1.4,            // Масштаб относительно ширины лица
        yOffset: 0.18          // Вертикальное смещение
    },
    glasses1: {
        type: 'glasses',
        url: './glasses1.glb',
        scale: 2.4,
        yOffset: 0
    },
    glasses2: {
        type: 'glasses',
        url: './glasses2.glb',
        scale: 2.56,
        yOffset: 0
    }
};
```

### Если нужно отрегулировать позицию модели

1. Откройте консоль (F12)
2. Найдите строку: `✓ Обновление 30:` (каждые 30 кадров)
3. Проверьте значения: `масштаб`, `угол`, `позиция`, `видимость`
4. При необходимости отрегулируйте в коде:
   - `scale` - для размера
   - `yOffset` - для вертикального смещения

## 🚨 Частые ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `NotAllowedError` | Разрешение отклонено | Нажать "Разрешить" в браузере |
| `NotFoundError` | Камера не найдена | Подключить камеру / перезагрузить браузер |
| `NotReadableError` | Камера занята | Закрыть другие приложения с камерой |
| Модель не видна | Слабое видео обнаружение | Улучшить освещение / подвинуться ближе |
| Модель падает за край | Калибровка | Отрегулировать `yOffset` и `scale` |
| CORS ошибка | Неправильные пути | Использовать `./` перед именем файла |

## 📊 Мониторинг производительности

Откройте вкладку Performance в консоли (F12 → Performance):

```
✓ Frame rate: 30-60 FPS (идеально 60)
✓ GPU usage: < 50%
✓ Memory: < 200MB
```

Если slow:
- Уменьшите разрешение камеры в `getUserMedia`
- Снизьте качество шейдеров Three.js
- Уменьшите количество обновлений лица (увеличьте интервал)

🔍 Проблема и решение

Исходная проблема:

Симптомы: Выпадающие списки меню отображаются позади основного контента страницы и не видны пользователям.

Причины:

  • Родительские контейнеры (header, header-inner, main-nav, dropdown) имели `height: 100%` или фиксированные высоты, создавая стекинг контексты
  • Отсутствовал `overflow: visible` на контейнерах, что ограничивало видимость выпадающих элементов
  • Z-index был установлен на dropdown-menu, но не на родительские контейнеры, создавая конфликты в иерархии слоёв
  • На десктопе фиксированная высота header в 80px приводила к обрезанию меню

Решение:

  • ✅ Добавлено `overflow: visible` на все контейнеры header (site-header, header-inner, main-nav, dropdown)
  • ✅ Установлено высокий z-index (2147483647 и 2147483646) на контейнеры для создания правильной иерархии слоёв
  • ✅ Изменена высота header с fixed значений на auto для позволения выпадающему меню переполнять границы
  • ✅ Добавлены transition и opacity для плавного отображения меню
  • ✅ Оптимизирована структура z-index для правильной работы на всех устройствах

📋 Важные заметки о CSS

Использованные CSS свойства:

  • `overflow: visible` - позволяет дочерним элементам выходить за границы родителя
  • `z-index: 2147483647` - максимальное значение z-index (2³¹ - 1)
  • `position: relative` - создаёт стекинг контекст для правильной работы z-index
  • `pointer-events: auto` - позволяет взаимодействовать с элементом (клики, наведение)
  • `visibility: visible` - показывает/скрывает элемент без влияния на layout
  • `opacity` - плавное появление/исчезновение с использованием transform

Почему это работает:

Правильная иерархия z-index:

  • `.site-header` - z-index: 2147483647 (самый высокий)
  • `.header-inner` - z-index: 2147483647
  • `.main-nav` - z-index: 2147483646
  • `.dropdown` - z-index: 2147483646
  • `.dropdown-menu` - z-index: 2147483647 (выше основного контента страницы)

Такая иерархия гарантирует, что выпадающие меню всегда будут видны поверх любого контента на странице.
